# Data Flywheel Map

## Flywheel Loops

### Loop 1: Correction
**Score: _2_/5**
*How does usage generate proprietary data?*
User interactions (role-play responses, retries, and adjustments) generate raw behavioral signals, but these are not yet systematically captured, labeled, or reused, so most of the learning signal is lost rather than turned into proprietary training data.

#### Improvement consideration: Capture and structure every user correction and expert override so feedback continuously improves coaching quality.

### Loop 2: Preference
**Score: _2.5_/5**
*How does that data improve the model?*
Some implicit signals (scenario choices, interaction patterns) can inform personalization, but without persistent user profiles or firm-level playbooks, the system only lightly adapts, limiting meaningful improvement in coaching relevance.

#### Improvement consideration: Build persistent user and firm profiles so the system adapts coaching style, tone and scenarios over time.

### Loop 3: Domain Context
**Score: _3_/5**
*How does the better model improve UX?*
A strong foundation of accounting scenarios and best-practice methodology improves realism and coaching quality, but because this knowledge is mostly static, improvements are not yet dynamically driven by real usage or outcomes.

#### Improvement consideration: Continuously enrich scenarios and playbooks with real-world patterns and outcomes to make coaching more relevant and realistic.

### Loop 4: Network
**Score: _1.5_/5**
*How does better UX drive more usage?*
Users operate in isolation, with no shared learning across individuals or teams—so improvements don’t compound across the network, limiting the impact on broader usage growth.

#### Improvement consideration: Turn individual interactions into shared insights so each user improves the system for everyone.

**Total Flywheel Score: _9_/20**
**Weakest Loop:**
Network
**Fix for weakest loop:**
Create shared learning mechanisms that allow insights from one user to benefit others, such as:

1. Aggregating and surfacing “what good looks like” across users
2. Building team-level dashboards and benchmarking
3. Capturing and distributing high-performing conversation patterns
4. Enabling manager feedback to improve coaching for all users

The goal is to move from isolated practice into collective intelligence, where each user interaction strengthens the system for everyone.

## 1. Platform Encroachment
Attacker: OpenAI (also Microsoft / Google)
Vector: Native AI coaching + role-play embedded in ChatGPT / Copilot / Workspace
Time-to-threat: 6–12 months
% of value at risk: 60–70%

Why: They can replicate your core mechanic (role-play + feedback) and distribute it instantly inside tools users already use.

Our defense:  Go deep, not broad and build accounting-specific coaching IP:
1. Advisory frameworks
2. Industry-specific scenarios
3. “What good looks like” in accountant conversations
4. Create structured evaluation + scoring models (not just responses)
5. Focus on outcomes, not interaction: “Did this improve client value?”

Winning position: “They (OpenAI) generate answers. We train better accountants.”

## 2. Vertical Competitor
Attacker: Karbon or Canopy (or a new AI-native accounting startup)
Vector: Purpose-built accounting training + advisory coaching deeply integrated into practice workflows
Time-to-threat: 12–24 months
% of value at risk: 50–60%

Why: They understand:

Accounting workflows
Client lifecycle
Real firm pain points

And can go deeper than a horizontal AI tool

Our defense: Own the learning + performance layer

Become the system of record for skill development:
1. Progress tracking
2. Benchmarking across teams
3. Certification / readiness signals
4. Build manager + firm tooling:
5. Coaching dashboards
6. Team performance insights
Move from:
“tool” → “training infrastructure”
Winning position:

“We don’t manage work—we make your team better at it.”

## 3. Adjacent Expansion
Attacker: Intuit
Vector: Adds “AI advisor coaching” into existing accountant ecosystem (QuickBooks, ProAdvisor, etc.)
Time-to-threat: 6–18 months
% of value at risk: 70–80%

Why:
They already own:

Distribution (massive accountant base)
Data (real SMB + accountant interactions)
Workflow (where work actually happens)

👉 They don’t need to be better—just integrated

our defense:

👉 Integrate, don’t compete head-on

Plug into their ecosystem:
CRM / practice tools
Accounting platforms
Position as:
Specialist layer, not replacement
Build cross-platform intelligence:
Insights they don’t unify
Patterns across tools and workflows
Winning position:

“We make your existing tools smarter and your people better.”
